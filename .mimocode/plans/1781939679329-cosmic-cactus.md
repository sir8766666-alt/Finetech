## Implementation Approach

### Edit UI Pattern (for items 1-3)
Each client component will add:
- `editingProject` / `editingIncome` / `editingExpense` state (null or item)
- Edit button per row â†’ sets state â†’ opens same dialog with pre-filled values
- Dialog title changes: "Create X" â†’ "Edit X"
- Submit handler checks: if editing, call `updateX(id, formData)`; else `createX(formData)`
- After submit: close dialog, clear editing state, `router.refresh()`

### Dashboard fix (item 4)
- Add `pendingPayables` computation alongside `pendingReceivables`
- Add 7th StatCard (grid will auto-wrap to 3 cols)

### Error handling (item 5)
- Wrap each `handleSubmit` and `handleDelete` in try/catch
- Display server errors via a `toast` or inline error state
